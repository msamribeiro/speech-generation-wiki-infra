# Generation Provenance Schema

Every Markdown page generated or substantively regenerated by an agent in the wiki content
repository carries a `generation` frontmatter block. This applies to Tier 1 paper pages, Tier 2
stubs, reviewed paper pages, concept Overviews, Concept In Depth pages, field overview pages, briefs, reports,
venue reports, and future generated page types. Existing version-1 blocks remain valid historical
records and do not need a bulk migration.

## Version 2

```yaml
generation:
  schema_version: 2
  date: "YYYY-MM-DD"
  runtime: claude-code | codex
  provider: anthropic | openai
  agent: speech-generation-{workflow}-agent
  model: "exact runtime model ID"
  commit: "7-character infra repository commit"
```

Render outputs also include `stage: render` and `mode: full | light`. Other page types may add
operation-specific fields without removing the required fields above.

- `runtime` records the execution surface, independently of the model vendor.
- `provider` records who provides the model. Use `anthropic` for Claude models and `openai` for
  OpenAI models.
- `model` is the exact model identifier exposed by the runtime. Never copy an example from these
  docs and never infer a more specific identifier than the runtime exposes. Use `"unknown"` only
  when the runtime does not expose an identifier.
- `agent` is the stable logical workflow name, shared across Claude Code and Codex.
- `commit` is the infra repository commit used to perform the operation, even though the page is
  written to the content repository.

Metadata `generation_history` entries use the same required provenance fields plus `op`. Changelog
entries for agent operations include `runtime`, `provider`, and `model` so a run can be identified
without opening every generated page.

Version-1 provenance is any historical block or history entry without `schema_version`. Preserve it
unchanged. Whenever a page is reviewed, re-ingested, or regenerated, replace its active frontmatter
block with version 2 and append a version-2 history entry where the workflow maintains history.
